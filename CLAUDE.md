# CLAUDE.md — 商品検索エージェント運用ガイド

このリポジトリは、日本の主要 EC サイト（Amazon.co.jp / メルカリ / 楽天市場 /
iHerb / オオサカ堂）を横断検索する AI エージェントの定義集です。
Claude Code は本ファイルの規約を**必ず**遵守してください。

---

## 0. 最重要：3 つの過去失敗パターンと絶対遵守事項

過去の商品検索で繰り返し発生した失敗。再発したらレポートを破棄して
やり直してください。

### 失敗パターン 1: 偽リンクの掲載（検索結果ページ URL を商品ページとして提示）

絶対 NG な URL 例：

    https://www.amazon.co.jp/s?k=...
    https://search.rakuten.co.jp/search/mall/...
    https://jp.mercari.com/search?keyword=...
    https://jp.iherb.com/search?kw=...
    https://osakado.org/search.html?...

対策：

- URL を掲載する前に、**個別商品ページ形式**であることを必ず検証する
- URL に以下の文字列を 1 文字でも含む場合は**即座に除外**：
  `search`, `?k=`, `?s=`, `?keyword=`, `?kw=`, `?q=`, `/search/`, `/s?`
- 個別商品 ID（ASIN / 楽天商品 ID / メルカリ item ID 等）が取得できない
  場合は「URL 取得不可」と記録し、候補から除外する

### 失敗パターン 2: 存在確認なしの紹介（実在する SKU を未確認で掲載）

「ブランドが実在する」と「その商品がそのチャネルに在庫ありで存在する」は
別問題として扱うこと。

対策：

- 一次情報（検索スニペットの本文・タイトル・価格・レビュー件数）で
  実在を必ず確認
- 「たぶん売ってそう」「一般的に売られている」での推測掲載は禁止
- 確認できない場合は「在庫確認不可」として候補から除外
- 最低限、以下のいずれかが取得できること：
  - 検索結果スニペット内に商品名と URL が明示されている
  - レビュー件数が表示されている
  - 価格が表示されている

### 失敗パターン 3: 条件の抜け漏れ（必須条件の未確認掲載）

対策：

- ユーザー入力から必須条件をリストアップし、漏れなく全項目チェック
- 各商品×各条件について `✅確認済み` / `⚠️要確認` / `❌不適合` を明記
- 1 つでも「要確認」「不適合」があれば**推奨 TOP3 から除外**
- 検索開始前に条件チェックリストをユーザーに提示し確認を取る

### 失敗パターン 4: 価格未報告 / 価格レンジ未比較

症状：レポートに「価格：要最新確認」「価格不明」が並び、ユーザーが
    実際の支出イメージを持てない。さらに中古・新古品・マーケットプレイス
    出品などの価格レンジ比較が漏れる。

対策：

- **価格は必須出力項目**。取得できなければ候補を ⚠️要確認に落とす
- スカウトエージェントは検索スニペット / 価格タグ / 通貨記号付き数値を
  最低 1 つは必ず取得する
- `¥XXX〜¥YYY` のレンジで良いので必ず数値を記載する
- レポート自己チェックに「TOP3 全商品に価格数値が入っているか」を追加
- 新品検索で候補を特定した後、**失敗パターン 4 の後段対策として
  必ず中古市場の並行検索を実施**（→ セクション 4 のフェーズ 2.5 参照）
- 価格は変動するため取得時刻を明記

## 1. 対象チャネルと URL 形式

### Amazon.co.jp

- 個別商品 URL：`https://www.amazon.co.jp/dp/[ASIN]`
- ASIN 形式：`[A-Z0-9]{10}`
- robots.txt により直接フェッチ不可。検索スニペット内から ASIN を抽出して
  URL を組み立てる
- 「Currently unavailable」「在庫切れ」は除外

### メルカリ

- 個別商品 URL：`https://jp.mercari.com/item/[ITEM_ID]`
- ITEM_ID 形式：`m[0-9]{10,}` または英数字の商品 ID
- 出品終了・売り切れ（SOLD）は除外
- 中古品のため**状態（新品 / 未使用 / 目立った傷なし 等）必須記載**
- 出品者評価が確認できる場合は付記

### 楽天市場

- 個別商品 URL：`https://item.rakuten.co.jp/[SHOP]/[ITEM_ID]/`
- 文字化け（EUC-JP）対応：URL 存在確認は検索スニペットで行う
- ショップレビュー / 商品レビューの両方を確認
- 「あす楽」対応の有無を付記

### iHerb

- 個別商品 URL：`https://jp.iherb.com/pr/[商品名スラッグ-ID]`
- サプリ・健康食品中心。**成分表記の確認が重要**
- 在庫状況（In Stock / Out of Stock）必須確認
- セール価格表示時は**通常価格も併記**

### オオサカ堂

- 個別商品 URL：`https://osakado.org/products/[ITEM_ID].html`
- 海外医薬品・ジェネリック中心
- **成分・用量の明記必須**
- 在庫状況と発送国を確認
- **個人輸入の注意事項を必ず付記**

### Yahoo!ショッピング（中古・並行輸入を含む価格比較に有効）

- 個別商品 URL：`https://store.shopping.yahoo.co.jp/[SHOP]/[ITEM_ID].html`
- マスター商品：`https://shopping.yahoo.co.jp/products/[PRODUCT_ID]`
- 検索結果 URL：`https://shopping.yahoo.co.jp/search?p=...` ← **絶対禁止**
- PayPay ポイント還元率を取得できれば付記
- 「ストア評価」「商品レビュー」を確認

### Yahoo!オークション（中古市場として利用）

- 個別商品 URL：`https://page.auctions.yahoo.co.jp/jp/auction/[ITEM_ID]`
- 検索結果 URL：`https://auctions.yahoo.co.jp/search/search?p=...` ← **絶対禁止**
- 即決価格・現在価格・入札数を取得
- 残り時間と終了済み区分を確認

### 追加チャネル（ユーザー指示時のみ）

Qoo10 / AliExpress / eBay 等。追加時は同一の 3 ゲート検証ルールを適用。

## 2. 3 段階の検証ゲート（全チャネル共通）

すべての候補商品は以下 3 ゲートを順に通過しなければレポートに掲載不可。

ゲート 1：URL 検証

- 個別商品ページ形式か？
- 禁止パターン文字列を含まないか？
- ID（ASIN 等）が有効な形式か？

ゲート 2：存在・在庫検証

- 一次情報で実在確認できたか？
- 在庫切れ・取り寄せ・入荷待ちでないか？
- レビュー件数 / 価格 / 商品名 のいずれか以上が取得できたか？

ゲート 3：条件充足検証

- ユーザー指定の必須条件を**全て**満たしているか？
- `✅` / `⚠️` / `❌` で明記したか？
- `⚠️` / `❌` を 1 つでも含む場合は推奨 TOP3 除外

---

## 3. 評価・価格に関するデフォルト値

- 星評価閾値：**4.0 / 5.0 以上**
- レビュー件数の信頼下限：**10 件**（10 件未満は慎重扱い）
- 価格基準：**セール・クーポン適用前の通常価格**
- 送料込み / 別を必ず明示
- 価格・在庫は変動する旨を必ず注記

### 3.1 価格は必須出力項目（失敗パターン 4 対策）

- 全候補について**価格の数値を必ず記録する**。取得できない候補は
  `⚠️要確認` 扱いにし TOP3 から除外
- レンジ可：`¥3,000〜¥4,500（送料別）` など
- TOP3 には必ず「取得時点の具体的な数値」を入れる
- 検索スニペットに価格がない場合は、別のクエリ（例：
  `"商品名" 価格` / `"商品名" 円`）で再検索する
- 価格取得の試行を 2 回行っても不明なら、レポートにその旨を明記し
  「価格未確定商品として TOP3 除外」と記録

## 4. ワークフロー（標準）

1. ユーザー要件確認 → 条件チェックリスト提示 → ユーザー承認取得
2. チャネル別スカウトエージェントを順次実行（並行処理可だが確実性優先）
3. 各候補に 3 ゲート検証を適用
4. **フェーズ 2.5：中古市場サーチ**（中古が成立する商品カテゴリの場合、必ず実施）
   - TOP3 候補が確定した後、その具体商品名で以下を再検索
     - メルカリ（個人間中古）
     - 楽天市場内の中古出品
     - Amazon マーケットプレイス中古 / Amazon Warehouse
     - Yahoo!ショッピング / Yahoo!オークション
   - 各チャネルで価格レンジと状態を取得し、レポートに「中古相場」セクションで掲載
   - 中古が成立しないカテゴリ（サプリ、医薬品、食品、衛生用品等）は
     本フェーズをスキップし「中古対象外」と明記
5. レポート出力エージェントが表形式（プレーンテキスト）レポートを生成
6. レポートを `reports/search_YYYYMMDD_HHMM.txt` に保存
7. `reports/index.txt` に履歴追記

### 中古が成立しない / 推奨しないカテゴリ例

- サプリ・ビタミン・プロテイン等の口に入れるもの
- 医薬品・医療機器
- 化粧品・衛生用品
- 下着・水着
- 開封済みで価値が落ちる消耗品

これらは中古サーチをスキップし、レポートに「中古対象外」と明記する。

---

## 5. レポート出力形式

- **プレーンテキスト**（Markdown 不使用）
- セクション見出しは `==` と `--`
- 箇条書きは `-`
- 階層はインデント（2 スペース）
- リンクは生の URL（角括弧不使用）
- 雛形は `reports/TEMPLATE.txt` を参照

---

## 6. レポート出力前の自己チェック（必須）

レポート確定前に以下を必ず確認：

- 掲載 URL に `search` `?k=` `?s=` `?keyword=` `?kw=` `?q=` を含まないか
- 各商品について実在・在庫の一次情報があるか
- 全必須条件について `✅` / `⚠️` / `❌` を明記しているか
- 推奨 TOP3 は全て `✅全充足` か
- **TOP3 全商品に価格の具体的な数値が入っているか**（失敗 4 対策）
- **中古対象カテゴリなら中古相場セクションが存在するか**（フェーズ 2.5 対策）
- 価格変動の注記があるか
- 取得時刻が記載されているか
- ファイル名がタイムスタンプ付きか

1 つでも欠けていればレポートを破棄してやり直し。

## 7. リポジトリ構成

    .claude/
      agents/             ... チャネル別スカウト・検証・出力エージェント
      commands/           ... カスタムスラッシュコマンド
    reports/
      TEMPLATE.txt        ... レポート雛形
      index.txt           ... 検索履歴インデックス
      search_*.txt        ... 個別検索レポート（タイムスタンプ付き）
    CLAUDE.md             ... 本ファイル

---

## 8. 行動規範

- 検索は必ずチャネルごとに分割して段階的に実行
- 各チャネル完了時に進捗を報告
- 不明点があれば作業前に質問
- 候補不足時は条件緩和を提案
- 推測での掲載は禁止
- ユーザーに選択を求める時は推奨と理由を必ず提示

# /search-product

商品検索エージェントのメインエントリポイント。以下のフェーズを順次実行する。

## 引数

$ARGUMENTS

引数が不足している場合は、以下を**先にユーザーに質問**してから着手すること：

- 商品名 / カテゴリ
- 必須条件（成分 / 容量 / 規格 / ブランド指定など、漏れなくリスト化）
- 予算の上下限
- 優先順位（価格 / 評価 / 配送速度）
- 対象チャネル（省略時はデフォルト 5 媒体）
- 除外条件

## 実行フェーズ

### フェーズ 1: 要件確認

1. ユーザー入力を受け、必須条件を箇条書きでリストアップ
2. 条件チェックリストをユーザーに提示して承認を取る
3. 対象チャネルと優先順位を確認

### フェーズ 2: チャネル別検索

以下のスカウトサブエージェントを順次実行する。並行実行してもよいが、
エージェントごとのタスクを独立させて確実性を優先すること。

- amazon-scout
- rakuten-scout
- mercari-scout
- iherb-scout
- osakado-scout

各チャネル完了時に進捗を報告すること。

### フェーズ 3: 3 ゲート検証

1. ゲート 1：url-validator サブエージェントで URL 検証
2. ゲート 2：各スカウトの出力から存在・在庫を確認
3. ゲート 3：condition-checker サブエージェントで必須条件充足を検証

全ゲートを通過した候補のみ「推奨 TOP3」の対象。

### フェーズ 4: レポート生成

1. report-writer サブエージェントを呼び、プレーンテキストの
   表形式レポートを生成
2. `reports/search_YYYYMMDD_HHMM.txt` として保存
3. `reports/index.txt` に履歴追記
4. 保存したファイルパスをユーザーに提示

### フェーズ 5: 自己チェック

- [ ] 掲載 URL に禁止パターン文字列を含まないか
- [ ] 一次情報で実在・在庫確認したか
- [ ] 必須条件を全項目チェックしたか
- [ ] 推奨 TOP3 は全て全充足か
- [ ] 価格変動注記があるか

## 禁止事項

- 検索結果ページ URL の掲載
- 推測での掲載（「たぶん売ってる」等）
- 必須条件の未確認のままでの推奨
- Markdown 記号の使用（出力は必ずプレーンテキスト）
